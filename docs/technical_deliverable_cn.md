# 图像用于针灸场景的高层与技术交付模板

本模板用于回答一个核心问题：

图像输入是否可以稳定、可控、可审计地支持针灸相关定位任务。

适用场景：

- 项目立项评审
- 中期技术评审
- 试点上线前验收
- 客户/管理层汇报

## 1. 交付目标

### 1.1 业务目标

- 目标任务：明确是穴位关键点定位、反射区分割，还是二者同时完成。
- 应用边界：明确仅用于辅助决策，不替代临床专业人员。
- 业务收益：明确效率提升、标准化收益、风险降低收益。

### 1.2 技术目标

- 精度目标：关键点误差、分割 Dice/IoU 达到预设阈值。
- 稳定性目标：不同人群、不同设备、不同光照下性能可接受。
- 可追溯目标：实验可复现，结果可审计，版本可回滚。

## 2. 成功标准（先定义，再实验）

在实验前固定“通过阈值”，避免结果导向。

### 2.1 关键点任务

- 平均像素误差（Mean Pixel Error） <= [阈值]
- 归一化距离误差 <= [阈值]
- PCK@[多个阈值] >= [阈值]

### 2.2 分割任务

- Mean Dice >= [阈值]
- Mean IoU >= [阈值]
- 最差类别 IoU >= [阈值]

### 2.3 工程指标

- 单张推理时延 <= [阈值] ms
- 显存占用 <= [阈值] GB
- 线上失败率 <= [阈值]

## 3. 数据与评测协议

### 3.1 数据划分

- 训练集：用于参数学习。
- 验证集：用于模型选择和调参。
- 测试集：仅用于最终报告。
- 外部测试集：来自不同来源或采集条件，用于泛化评估。

### 3.2 标注质量控制

- 标注规范文档版本：[版本号]
- 双人复核比例：[比例]
- 一致性指标（如 Dice/坐标差）：[数值]

### 3.3 数据覆盖分析

- 人群覆盖：年龄段、肤色分布、左右手比例。
- 场景覆盖：光照、遮挡、角度、距离。
- 设备覆盖：相机型号、分辨率、压缩方式。

## 4. 基线与消融（证明方案有效）

### 4.1 基线模型

- 基线 A（简单 CNN/热图法）：[结果]
- 基线 B（单任务分割）：[结果]
- 当前模型（多任务）：[结果]

### 4.2 消融实验

- 去掉先验提示（prompt）后性能变化。
- 仅关键点任务 vs 多任务联合训练。
- 输入分辨率变化（256/512）对性能影响。
- 训练数据规模（25%/50%/100%）对性能影响。

### 4.3 统计显著性

- 给出 95% 置信区间。
- 与基线做显著性检验（如 bootstrap 或 t-test）。

## 5. 鲁棒性与安全性

### 5.1 压力测试

- 模糊、噪声、压缩、低照度、偏色。
- 姿态极端变化、遮挡比例增加。
- 近距/远距拍摄变化。

### 5.2 不确定性与拒识策略

- 为每个关键点输出置信度或不确定度。
- 当低于阈值时拒绝自动输出，转人工确认。
- 记录拒识率与误拒率。

### 5.3 人机协同机制

- 模型仅给出建议结果。
- 专家确认后才能进入下一流程。
- 所有关键步骤保留日志与截图。

## 6. 工程化交付项

### 6.1 可复现性

- 固定随机种子与配置文件。
- 记录依赖版本、CUDA、驱动版本。
- 提供一键训练、一键评测命令。

### 6.2 产物清单

- 模型权重（带版本号）
- 配置文件
- 训练日志
- 评测报告
- 错误案例集
- 可视化样例

### 6.3 运行与部署

- 目标硬件性能报告（GPU/CPU）
- 峰值并发与吞吐测试
- 回滚方案与版本切换策略
- 线上监控指标（延迟、置信度分布、漂移）

## 7. 风险与限制说明

必须明确写出“不能做什么”。

- 不能替代临床诊断。
- 对严重遮挡和极端姿态仍有失败风险。
- 训练数据覆盖范围之外存在泛化风险。

## 9. 附录：最小可交付报告结构

建议把正式报告控制在以下结构：

1. 执行摘要（1页）
2. 任务定义与业务价值
3. 数据与标注
4. 模型与方法
5. 实验协议
6. 结果与对比
7. 失败案例与风险
8. 工程化与上线建议
9. 结论与路线图

---

可直接用于你当前仓库的落地建议：

- 先使用小数据做流程自检，再用完整数据集做正式评估。
- 先输出“技术可行性结论”，再申请“业务试点”。
- 所有指标以测试集和外部测试集共同结论为准。
